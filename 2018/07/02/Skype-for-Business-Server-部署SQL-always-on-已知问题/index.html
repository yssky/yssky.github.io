Skype for Business Server 部署SQL always on 已知问题

发表于 2018-07-02 | 分类于 Skype for Business Server

Skype for Business Server 2015后端SQL数据库部署Always On后，SQL数据库可用性组和可用性数据库已经准备就绪。但是当故障转移至辅助副本时，问题就出现了，前端频繁报错：无法连接至后端数据库，用户Skype for Business客户端报“受限的连接”，无法获取联系人列表。

解决方法，微软宣讲的Lync升级Skype for Business PPT中有讲到此已知问题，如下图

具体的解决步骤：

1.打开群集管理器手动故障转移，切换至副本节点；或打开SQL控制台，故障转移至辅助副本；

2.以管理员身份打开Skype for Business Server 命令行管理程序，运行如下命令更新辅助副本登录用户信息

其中xxxsql.xx.cn填写SQL Always On侦听器地址

Install-CsDatabase -Update -ConfiguredDatabases -SqlServerFqdn xxxsql.xx.cn -Verbose
