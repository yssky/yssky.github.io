Skype for Business Server频繁报错LS Data MCU 事件ID:41026

发表于 2018-07-02 | 分类于 Skype for Business Server

Skype for Business Server 2015部署前习惯把Windows Server操作系统补丁打到最新，最近在部署完毕后，Lync Server 事件中频繁报错事件ID：41026（LS Data MCU）和正常的状态信息41205，具体报错信息如下图：

此故障是已知问题，因为安装.NET Framework 4.5.2 或最新版本中会在增强型密钥使用（EKU）上向证书添加额外的检查，由于颁发证书的模板中没有添加客户端验证造成的。

具体的解决方法有两种，详见如下链接：

https://support.microsoft.com/en-us/help/4023993/ls-data-mcu-events-41025-and-41026-are-constantly-generated-after-you-

https://blogs.technet.microsoft.com/uclobby/2017/05/24/lyncsfb-server-event-41026-ls-data-mcu-after-may-2017-net-framework-update/

此次我使用第二种方法，关闭EKU证书检查，在注册中添加键值关闭EKU检查。

具体解决步骤如下

1.导航到如下图注册表位置，如果不存在项：System.Net.ServicePointManager.RequireCertificateEKUs，需要手动创建，过程如下图

2.下载自动判断Lync/Sfb Server版本Powershell脚本https://gallery.technet.microsoft.com/LyncSfB-Server-Disable-EKU-dab6cb88，自动创建键值关闭EKU检查，当然也可以手动创建，此次使用脚本的方式（适用于Sfb Server安装位置为默认位置），以管理员命令打开，过程如下图

3.核实注册表键值是否创建

4.重新启动Skype for Business Server Web会议服务

至此，再通过事件查看器，查看已经没有此报错信息，前端与边缘的Web会议正常建立连接。
